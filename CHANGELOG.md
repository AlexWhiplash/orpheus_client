## [Unreleased]
### Added
- **Поиск по контактам** (`PROD-3`): иконка поиска на вкладке «Контакты» открывает строку фильтра по имени (в памяти, без сети). При активном поиске показываются только совпадения (Оракул и подсказка скрыты), при пустом результате — «Контакты не найдены». Раньше при росте списка оставался только скролл. Строки через L10n (EN/RU).
- **Онбординг при первом запуске** (`PROD-6`): после создания аккаунта новый пользователь один раз видит короткую подсказку из 3 шагов — «ваша личность = ключ (поделитесь QR/ID)», «добавьте контакт (скан QR или вставка ключа)», «защитите приложение PIN-ом». Показывается один раз (гейт по `SharedPreferences`, как у бета-дисклеймера), двуязычно (EN/RU). Раньше новый пользователь попадал на почти пустой экран без объяснения, как подключиться.
- **Шифрование локальной БД на устройстве (SQLCipher).** Вся база (сообщения, контакты, заметки, AI-переписка) шифруется на диске ключом из Android Keystore (`flutter_secure_storage`). Закрывает критическую находку аудита `SEC-1`: теперь данные недоступны при изъятии/руте/облачном бэкапе телефона — PIN/duress/wipe перестают быть косметикой. Ключ БД удаляется при panic-wipe (остаточный шифртекст невосстановим).
- CI (GitHub Actions): `flutter analyze` (жёсткий гейт на ошибки) + `flutter test` на пуш в ветки `wl/**` и на pull request.

### Changed
- **В строке контакта — время последнего сообщения вместо хвоста ключа** (`PROD-4`): список контактов уже сортировался по свежести (как инбокс), но в строке показывался бесполезный хвост публичного ключа. Теперь показывается компактное локализованное время последнего сообщения (сегодня — часы, иначе дата), а у контактов без переписки строка времени скрыта. **Приватность:** намеренно показываем только время, БЕЗ текста сообщения — содержимое не светится на главном экране (согласуется с тем, что уведомления скрывают контент, и с duress-режимом).
- **Пуши без Google: FCM → постоянный foreground-сервис + WebSocket.** Раньше входящие звонки/сообщения при убитом приложении будил Firebase Cloud Messaging (через серверы Google, с утечкой метаданных доставки). Теперь эту роль играет собственный **постоянный foreground-сервис** (`PushConnectionService` на базе `flutter_background_service`, тип Android `specialUse`), который держит WebSocket-соединение в отдельном isolate и показывает CallKit/локальные уведомления теми же проверенными функциями. Координация с UI: main-изолят пишет heartbeat; пока приложение живо — сервис молчит (его сокет не открывается, всё показывает UI), как только приложение убито — сервис берёт доставку на себя. Пересечения снимаются прежним дедупом по `call_id`/`message_id`. Микрофон во время звонка держит отдельный короткоживущий нативный сервис типа `microphone` (`CallAudioService`), который стартует из видимого экрана звонка — поэтому разговор продолжается, даже если приложение свернуть во время звонка (постоянный `specialUse`-сервис сам микрофон держать не может из-за правил Android 14, поэтому вынесен отдельный сервис — паттерн Signal/Molly). **Важно для надёжности:** доставка при убитом приложении требует исключения из энергосбережения и (на Xiaomi/Huawei/Oppo/Vivo) ручного автозапуска — программного аналога FCM-льготы не существует; эта часть требует проверки на реальных устройствах. Контракт бэкенда: WS-сообщение `register-fcm` больше не шлётся — сервер должен перестать полагаться на FCM-токены и доставлять оффлайн-события в WS-сокет (см. AI_WORKLOG для деталей).
- Зависимость БД: `sqflite` → `sqflite_sqlcipher` (тот же API + шифрование). При первом запуске зашифрованной версии старая НЕзашифрованная БД удаляется — перенос данных не выполняется (в приложении нет критичных данных).
- У сообщений появился стабильный `message_id` (UUID), одинаковый у отправителя и получателя; передаётся в конверте чата и «удалить у обоих». Обратно совместимо: старые клиенты игнорируют новое поле, а при отсутствии id используется прежнее окно дедупа. Схема БД поднята до версии 7. Задействован пакет `uuid` (ранее объявлен, но не использовался — `DEP-10`).
- JDK для Gradle резолвится через flutter/JAVA_HOME, без жёстко прописанного пути.
- Все локальные уведомления (звонок, сообщение, сообщение в чате, официальный ответ, тестовое, имя по умолчанию), CallKit-подписи входящего звонка (Ответить/Отклонить/Пропущенный звонок/Перезвонить) и подписи панели звонка (Микрофон/Динамик/Завершить) — **двуязычные (EN/RU)** через gen-l10n, без жертвы английским. Панель звонка берёт строки из `L10n.of(context)`; уведомления идут из фонового сервисного изолята без `BuildContext`, поэтому язык резолвится по сохранённому выбору (`app_locale`) → системная локаль → английский. Добавлены ключи l10n: `answerCall`, `microphone`, `newMessage`, `unknownCaller`, `missedCall`, `callBack`, `newMessageInChat`, `officialOrpheusReply`, `testNotificationWorks`, `fromCaller`. (Системные *названия* Android-каналов оставлены как есть — регистрируются один раз.)
- Экран «История обновлений» локализован: заголовок берётся из ключа `updateHistory` (`ИСТОРИЯ ОБНОВЛЕНИЙ` / `UPDATE HISTORY`) вместо захардкоженного английского текста.

### Performance
- Добавлен отдельный индекс `idx_messages_timestamp` на `messages(timestamp)` (`PERF-2`): авто-очистка истории фильтрует по `timestamp < ?` без `contactPublicKey`, и составной индекс `(contactPublicKey, timestamp)` ей не помогал — был полный скан таблицы. Схема БД поднята до версии 8 (миграция добавляет индекс существующим базам). Заметно легче при большой истории.
- Загрузка бейджей контактов дедуплицирует одновременные запросы (`PERF-4`): если бейдж для одного ключа уже запрашивается, параллельные вызовы переиспользуют тот же запрос вместо новых сетевых обращений (список контактов дёргает предзагрузку на каждое обновление).
- Чат больше не перечитывает всю переписку с диска на каждое входящее сообщение (`PERF-1`): новые сообщения дописываются инкрементально (только то, что новее последнего известного, с дедупом по `message_id`). Заметно легче для активных чатов с большой историей. _(Пагинация начальной загрузки — скролл вверх — остаётся отдельным follow-up.)_
- Список контактов не пере-агрегирует всю таблицу сообщений и не перерисовывается на каждое входящее (`PERF-3`): обновление коалесцируется debounce-ом (400 мс), что снимает лаг при пачке сообщений (напр. бэклог на реконнекте).

### Fixed
- **Кнопка «…» в чате больше не очищает историю в один тап** (`PROD-5`): раньше иконка меню сразу открывала подтверждение очистки истории (вводило в заблуждение). Теперь это настоящее меню: «Информация о контакте» (показывает полный публичный ключ для **сверки против MITM** по доверенному каналу + копирование) и «Очистить историю» (с прежним подтверждением). Строки через L10n (EN/RU).
- **Тумблер биометрии теперь настоящий** (`PROD-2` / `QUAL-3`): раньше он показывал «биометрия включена», но ничего не сохранял (в теле стоял `// TODO`), и на перерисовке отскакивал в «выкл». Теперь при включении (после подтверждения отпечатком/лицом) флаг `isBiometricEnabled` реально сохраняется в `AuthService`/`SecurityConfig`, при выключении — сбрасывается, а экран блокировки уже умел предлагать вход по биометрии по этому флагу. Приложение безопасности больше не «врёт» об включённой защите.
- Добавлен таймаут (watchdog) на установку WebSocket-соединения: если `connect` зависает, приложение больше не застревает навсегда в состоянии «Подключение» — по таймауту (20 с) попытка отбрасывается (зависший сокет закрывается через механизм «поколений»), происходит ротация хоста и переподключение.
- Устранена гонка при переподключении WebSocket, из-за которой на нестабильной сети (переключение Wi-Fi/сотовой) могли появиться **два живых сокета** — с двойной доставкой входящих сообщений и утечкой соединения (`LOGIC-9`). Введён «токен поколения» подключения: если во время установки соединения стартует новое, устаревший сокет закрывается и не подписывается; события закрытия устаревшего сокета игнорируются (иначе они запускали лишний цикл переподключений при живом соединении); намеренный `disconnect` тоже инвалидирует зависшую попытку. Плюс защита от параллельного слива очереди отложенных сообщений (не допускает дублей/потерь при быстром реконнекте). _Изменение проверено адверсариал-верификацией (2 линзы), найденный регресс с «дребезгом» переподключения исправлен._
- Сообщения от неизвестного отправителя (ещё не в списке контактов) больше не «пропадают» из виду (`DB-6`): при входящем сообщении от нового ключа контакт создаётся автоматически (имя по умолчанию — префикс ключа, можно переименовать), и переписка появляется в списке чатов. Раньше такое сообщение сохранялось в БД, но нигде не отображалось — пользователь видел только всплывающее уведомление.
- Офлайн-запуск больше не запирает уже активированного пользователя на экране лицензии (`LOGIC-8`): кэшируется последний подтверждённый сервером статус лицензии, и при отсутствии сети приложение открывается сразу по кэшу — **без ограничения по времени** (grace-период убран). Модель простая: лицензия проверяется на сервере, онлайн-отзыв применяется мгновенно (WS выставляет статус `false` и перезаписывает кэш), а без сети приложение доступно. Убран учёт времени последней проверки — меньше вычислений/обращений к часам, ничего не «тикает» в фоне.
- QR-сканер: при отказе в доступе к камере / сбое инициализации показывается понятное сообщение и кнопка «Открыть настройки» вместо чёрного тупика с анимацией (`UI-3`).
- Диалог добавления контакта: кнопка блокируется на время сохранения, добавлена валидация (пустые поля и формат публичного ключа — base64 от 32 байт) с инлайн-ошибкой и обработкой сбоя (`UI-4` / `UI-7`).
- Экран приветствия: кнопка «Создать аккаунт» блокируется и показывает индикатор на время генерации ключей, а сбой теперь показывается пользователю (`UI-5`). Раньше двойной тап мог перегенерировать ключи поверх только что созданных, а ошибка не давала обратной связи.
- **Статус исходящих сообщений теперь реальный** (`LOGIC-3` / `LOGIC-4` / `UI-8`): раньше всегда показывалась двойная галка «доставлено» — даже если шифрование или отправка упали (ошибка молча проглатывалась, и потерянное сообщение выглядело доставленным). Теперь статус отражает реальность: часы (отправляется) → одиночная галка (передано в сеть/поставлено в offline-очередь) → красный значок ошибки при сбое. Двойная галка «доставлено» пока не используется — сервер это E2E-релей без квитанций о доставке.
- **Тихая потеря входящих сообщений** устранена (`LOGIC-1`): дедупликация теперь по стабильному `message_id` (UUID в конверте), а не по грубому 5-секундному окну, из-за которого терялись разные сообщения от одного контакта, пришедшие подряд в пределах 5 секунд.
- **«Удалить у обоих» теперь срабатывает у получателя** (`LOGIC-2`): удаление по `message_id`, а не по метке времени (у отправителя и получателя метки различались из-за задержки/расхождения часов, и удаление не находило строку).
- БД (`DB-4`): убран UNIQUE-индекс по `(contactPublicKey, timestamp, isSentByMe)`, который мог молча терять сообщения с одинаковой миллисекундой; заменён на UNIQUE-индекс по `(contactPublicKey, messageId)` (дедуп) + обычный индекс по `(contactPublicKey, timestamp)` (под запросы чата).
- Компиляция на актуальном Flutter (3.44.x): `CupertinoPageTransitionsBuilder` импортируется из `package:flutter/cupertino.dart` (перестал реэкспортироваться из `material`).
- Тесты: устранены все 20 падений на Flutter 3.44 → **324 passed / 0 failed** (был 303/20). Большинство — version-skew (устаревшие RU-литералы в ассертах, забытая локаль в pump'ах), плюс реальные дефекты ниже.
- Карточка меню настроек: ListTile'ы обёрнуты в `Material(type: transparency)` — убран debug-ассерт Flutter 3.44 «ListTile background color or ink splashes may be invisible» и возвращён видимый ripple при нажатии на пункты.
- Язык диалогов бета-дисклеймера (главный экран) и подтверждения wipe (экран блокировки) берётся из виджет-дерева (`Localizations.localeOf(context)`), а не из глобального синглтона `LocaleService`, — диалоги корректно следуют за локалью приложения (в проде поведение не меняется).
- Тест `getWithFallback` переписан под текущую реальность одного API-хоста (legacy-домен убран ради приватности): проверяются успех основного хоста и возврат `null`, когда все хосты недоступны.

### Security
- **Де-гуглизация (курс на клиент без сервисов Google).** Первый пакет privacy-правок:
  - Шрифт Inter **забандлен в приложение** (статические TTF w400–w700, лицензия OFL в `assets/fonts/OFL.txt`), пакет `google_fonts` удалён. Раньше шрифты могли загружаться в рантайме с серверов Google (`fonts.gstatic.com`). Типографика теперь полностью офлайн.
  - **Google Analytics for Firebase удалён.** Убрана нативная зависимость `com.google.firebase:firebase-analytics` (+ `firebase-bom`) из `android/app/build.gradle.kts` — она подключалась напрямую, из Dart-кода **не использовалась вообще**, но по умолчанию слала события (first_open, session_start, идентификатор устройства) на `app-measurement.com`. Никакого кода приложения это не затрагивает; FCM тянет свои зависимости сам.
  - **Sentry — строго opt-in.** Сторонний краш-репортер (`ingest.de.sentry.io`) раньше инициализировался безусловно при каждом запуске и слал крэши/трейсы наружу без согласия. Теперь `SentryFlutter.init` вызывается только если пользователь явно включил телеметрию (тот же флаг `telemetry_enabled`, что у `TelemetryService`); по умолчанию наружу не уходит ничего. `environment` берётся из `kReleaseMode`, а не хардкодится в `production`.
  - **Утечка региона через сторонний IP-сервис устранена** (`ARCH-6`): экран «Система» больше не делает plaintext-запрос к `http://ip-api.com/json/` (отдавал IP пользователя третьей стороне по незашифрованному HTTP при каждом открытии). Регион теперь определяется по локали устройства, без единого сетевого запроса.
  - **Фоновая телеметрия приведена к opt-in** (регресс `SEC-2`): путь `notification_service._sendBackgroundTelemetry` (срабатывал на фоновых событиях звонка) слал `peer_pubkey` (ключ контакта = граф общения) и сырой payload на `/api/logs/batch` **в обход** флага телеметрии и санитизации. Теперь он уважает `telemetry_enabled` и не отправляет идентификаторы контактов.
  - **QR-сканер снят с Google ML Kit.** Пакет `mobile_scanner` (тянул закрытый бинарник `com.google.mlkit:barcode-scanning`, ~3 МБ, отправлявший usage-метрики в Google) заменён на `qr_code_dart_scan` — декодирование QR на **чистом Dart** (`zxing_lib`, BSD-3) поверх официального `camera`-плагина. Ноль зависимостей от Google Play Services / ML Kit.
  - **Firebase Cloud Messaging полностью удалён** (см. раздел «Пуши без Google» ниже). Убраны `firebase_core`, `firebase_messaging`, плагин `com.google.gms.google-services`, `google-services.json`, firebase-правила ProGuard и FCM-meta-data из манифеста. В сборке APK больше **нет ни одного артефакта Google/GMS/Firebase**.
- Настроена подпись **release-сборки собственным keystore** через `android/key.properties` (`OPS-1`): если файл настроек есть — release подписывается вашим ключом, иначе идёт фолбэк на debug-ключ (сборка и CI работают без секретов). Добавлены `android/key.properties.example` с инструкцией генерации keystore и правила `.gitignore` (ключ и пароли не попадают в git). Раньше release подписывался debug-ключом — такой сборкой нельзя обновлять приложение в сторе.
- **`applicationId` сменён с `com.example.orpheus_project` на постоянный `click.orpheus.app`** (`OPS-2`): reverse-DNS домена проекта (orpheus.click). `com.example.*` банится сторами, а сам id после первого публичного релиза сменить нельзя — поэтому зафиксирован до релиза. Внутренний namespace/package намеренно оставлен прежним (магазину не виден, переименование дало бы лишний риск ради косметики). Смена стала тривиальной после удаления Firebase — больше не нужно перерегистрировать google-services.
- PIN и коды принуждения/удаления теперь хэшируются **настоящим Argon2id** (memory-hard), а не 10000×SHA-256 под видом Argon2id (`SEC-8`). Подбор PIN по украденному конфигу стал на порядки дороже (Argon2id требует памяти, что душит перебор на GPU/ASIC). Совместимость: PIN, заданные до обновления, продолжают работать — проверяются по старой схеме и **автоматически апгрейдятся на Argon2id при первом успешном входе** (без сброса и потери данных).
- Счётчик неудачных попыток PIN теперь записывается **надёжно (await) до реакции интерфейса** (`LOGIC-5`): форс-килл приложения между вводом и сохранением больше не сбрасывает прогресс блокировки и авто-wipe.
- Аварийный wipe (panic / auto-wipe / wipe-код / удаление аккаунта) теперь **исчерпывающий, надёжный и best-effort** (`SEC-5` / `LOGIC-7` + `ARCH-7`): очищается ВСЁ secure storage целиком (`deleteAll`), а не заданный список ключей (раньше оставались сессия desktop-link с токеном и ключ шифрования БД); каждый шаг в своём `try/catch`, поэтому сбой одного не отменяет остальные, а сброс состояния/навигации выполняется всегда. По результатам адверсариал-проверки дополнительно исправлено: `CryptoService` стал синглтоном, и wipe теперь очищает ключи **у живого экземпляра** (раньше стирался одноразовый экземпляр, а приватный ключ оставался в памяти, и сокет — подключённым под стёртой личностью); сеть (`websocket`) разрывается **в начале** wipe, чтобы входящее сообщение не пересоздало БД+ключ во время очистки.
- В release-сборке структурированные логи (`DebugLogger`) больше **не пишутся в системный logcat** (гейт `kDebugMode`) — они остаются только в in-app экране отладки (RAM-буфер) и опциональной телеметрии. Дополнительно чувствительные события безопасности переведены с голого `print` на `DebugLogger`: убран дамп security-конфига (с хэшами PIN/duress/wipe и солью), события «duress/wipe код введён», «неверный PIN», авто-wipe, а также префикс публичного ключа и состояние PIN при старте. Закрывает основную часть `QUAL-1` / `OPS-6` (утечка событий безопасности и метаданных в logcat на рутованном устройстве / в bug-report). Полный перевод остальных ~200 операционных `print` — отдельный follow-up (`QUAL-7`).
- Android Auto Backup выключен: `android:allowBackup="false"` + правила `res/xml/data_extraction_rules.xml` (исключают всё из облачного бэкапа и device-to-device переноса). Незашифрованная (и теперь зашифрованная) БД больше не покидает устройство через бэкап Google/adb. Закрывает `SEC-7` / `DB-2` / `OPS-3`.
- Телеметрия теперь **выключена по умолчанию** (opt-in): включается вручную из скрытого экрана отладочных логов. Из выгрузки убраны публичные ключи контактов (`peer_pubkey` и прочие идентификаторы личности) и отпечаток устройства (`device_info`); контекст логов очищается от контактных ключей перед отправкой. Закрывает `SEC-2`.

### Removed
- **Desktop Link удалён из клиента** (`PROD-1` / `ARCH-5` / `SEC-9`): паринг телефон↔десктоп по LAN (экран, сервис, LAN WebSocket-сервер, модель сессии, l10n, тест — ~690 строк) был недостижим из UI (мёртвый код в каждой сборке) и держал небезопасный протокол: `session_token`/`otp` слались открытым HTTP, WS-сервер слушал `anyIPv4:8765` без проверки токена, а `desktop_pubkey` из QR не использовался (канал не шифровался). Удалён целиком; вернём безопасно после доработки клиента и сервера, когда десктоп-приложение дозреет (история сохранена в git, план — в памяти проекта). panic-wipe и так уже чистил остаток сессии (`deleteAll`).
- Мёртвый код (аудит `ARCH-4` / `QUAL-2` / `DEP-9`): удалены `lib/main_test.dart` (осиротевшее WebRTC-тест-приложение со своим `runApp()` внутри `lib/`, ни на что не ссылалось), пустые файлы-заглушки `call_session_controller.dart` / `chat_session_controller.dart`, и неиспользуемая зависимость `provider`. Меньше шума и поверхности для поддержки.
- Машинно-специфичный `org.gradle.java.home` из `android/gradle.properties` (был жёстко прописан путь другого разработчика, ломал сборку на других машинах).
