# Device-чеклист перед релизом (накоплено за сессию рефакторинга)

Всё ниже — то, что **нельзя проверить сборкой/тестами**, только на реальном
устройстве. Правило владельца: **не выпускаем в релиз, пока всё не победим.**

Собрать **release**-APK (`flutter build apk --release`, подписан реальным
keystore — OPS-1) и гонять на матрице устройств. Матрица (квирки кластеризуются
по OEM и версии Android):

- [ ] Pixel / чистый Android
- [ ] Samsung (One UI)
- [ ] Xiaomi (MIUI/HyperOS)
- [ ] Android 13, 14, 15 (разные)

> **minSdk поднят 23 → 24: Android 6.0 больше НЕ поддерживается.** Проверить, что
> приложение ставится на Android 7+ и отсутствует в сторе для Android 6.

---

## 1. Secure storage v10 (чистая установка)

Все ставят с нуля (решение владельца) → тест обновления v9→v10 поверх **пропущен**,
старый APK не нужен. Проверяем только путь чистой установки.

- [ ] Пройти раздел **«Чистая установка»** в
  [SECURE_STORAGE_V10_CHECKLIST.md](SECURE_STORAGE_V10_CHECKLIST.md): создание
  аккаунта на современных шифрах, вход по PIN после перезапуска и **после ребута
  устройства** (у v10 исторически ломалось «чтение после boot»), panic-wipe.

## 2. Звонки — мажоры плагинов + вынос логики в CallSessionController

Ядро, переписанное в этой сессии (callkit 2→3, перевязка на контроллер, 2
пойманных ревью регресса реконнекта). Проверять придирчиво.

- [ ] **Исходящий звонок**: соединяется, слышно в обе стороны.
- [ ] **Входящий звонок (приложение открыто)**: CallKit-экран, приём, звук.
- [ ] **Входящий звонок из УБИТОГО состояния**: доставка через push-сервис
  (specialUse FGS, без FCM), CallKit поверх локскрина, приём, звук.
- [ ] **Full-screen входящего с локскрина** (enableCallMode).
- [ ] **Микрофон при свёрнутом приложении во время разговора** (нативный
  CallAudioService FGS, Android 14 while-in-use) — собеседник продолжает слышать.
- [ ] **Реконнект при смене сети Wi-Fi ↔ cellular во время звонка** — звонок НЕ
  залипает на «Reconnecting...», аудио восстанавливается. *(Здесь ревью нашло 2
  бага — проверить особенно тщательно: (а) реконнект не умирает после 1 попытки;
  (б) при исчерпании попыток экран авто-закрывается.)*
- [ ] **Исчерпание реконнекта**: после серии неудач экран показывает «Failed» и
  **сам закрывается** через ~2с.
- [ ] **Собеседник кладёт трубку** → экран закрывается, статус-сообщение в чате.
- [ ] **Отклонение/отмена** входящего/исходящего.

## 3. Уведомления (flutter_local_notifications 17→22)

- [ ] Запрос разрешения на уведомления (Android 13+).
- [ ] Показ уведомления о новом сообщении, тап открывает нужный чат.
- [ ] Каналы уведомлений созданы корректно (звонок/сообщение).
- [ ] Уведомление о звонке скрывается при открытии экрана звонка.

## 4. Биометрия и PIN (local_auth 2→3, Argon2id PIN — SEC-8)

- [ ] **Разблокировка по биометрии** (отпечаток/лицо) на локскрине и в настройках.
- [ ] **Вход по PIN** (Argon2id, verifyPin теперь async) — задержки нет заметной.
- [ ] **Duress-код** → пустой профиль; **wipe-код** → подтверждение и стирание.
- [ ] **LOGIC-6:** после серии неверных PIN включается блокировка (30с/1м/...).
  **Перевести системные часы вперёд — блокировка НЕ снимается** (монотонные часы).

## 5. De-Google (полное удаление сервисов Google)

- [ ] Шрифты рендерятся (bundled open-source, без Google Fonts из сети).
- [ ] Доставка сообщений/звонков в убитом состоянии работает **без** Google Play
  Services (проверить на устройстве/эмуляторе без GMS, напр. чистый AOSP).
- [ ] (Уже проверено в сборке: release-APK не содержит classes
  com.google.firebase/mlkit/gms — при желании перепроверить `dexdump`.)

## 6. Лицензия (убран таймаут кэша)

- [ ] **Оффлайн** (нет интернета) → приложение доступно, не блокируется.
- [ ] (Требует сервера) Отозванная лицензия → приложение отключается при онлайне.
