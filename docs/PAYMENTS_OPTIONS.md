# Orpheus — покупка лицензии в приложении: варианты и решения

Детальная проработка пункта 3 роадмапа. Для владельца (решение) и кодера (реализация).

---

## 0. Рамки и ключевая идея

**Ограничения концепции:**
- Прямой APK, НЕ Google Play → Google Play Billing и Apple IAP отпадают.
- Приватность/анонимность/де-Google → без карт, KYC, имени, привязки к личности.
- Лицензия привязывается к **публичному ключу** пользователя (это и есть его
  идентичность в системе), НЕ к телефону/почте/имени.
- Нужно: **просто и быстро**.

**Уже есть на бэкенде:** активация по коду (`POST /api/activate-promo`) + статус
лицензии прилетает по WS (`license-status: active`). Это фундамент.

**КЛЮЧЕВАЯ ИДЕЯ (архитектура):** способ оплаты отделён от выдачи лицензии. Любой
оплаченный путь сводится к одному: сервер помечает лицензию активной для нужного
pubkey (или выдаёт код). Клиент просто ждёт `license-status: active`. Это позволяет
добавлять/менять способы оплаты, не трогая клиента.

**Авто-активация без ручного кода (важно для UX):**
1. Приложение генерирует одноразовый `session_token` и показывает/передаёт его в
   путь оплаты (в диплинк бота / в metadata крипто-инвойса).
2. Пользователь платит.
3. Платёжный webhook на бэкенде связывает `session_token` → pubkey → активирует
   лицензию.
4. Приложение поллит `/api/license-status` (или ждёт WS-событие) → сама активируется.
Пользователю НЕ нужно копировать код руками. (Ручной код остаётся как запасной путь.)

## 1. Вариант A — Telegram-бот: TON / Stars  [самый быстрый UX]

**Как работает:**
- В приложении кнопка «Купить лицензию» → диплинк на нашего бота
  `t.me/orpheus_pay_bot?start=<session_token>`.
- Бот показывает тарифы и принимает оплату:
  - **TON** — через встроенный кошелёк Telegram (@wallet) или TON Connect. Нативно,
    дёшево, быстро, полу-анонимно.
  - **Telegram Stars** — родная валюта Telegram (`sendInvoice`, валюта `XTR`).
- После оплаты бот-бэкенд связывает `session_token` → активирует лицензию → приложение
  само подхватывает статус (или бот присылает код запасным путём).

**Клиент:** одна кнопка + диплинк + поллинг статуса. Мало работы.
**Сервер:** Telegram-бот (aiogram/python-telegram-bot) + обработчик успешной оплаты →
привязка token→pubkey→лицензия.

**Плюсы:** быстрее и проще всего для пользователя (аудитория приватных мессенджеров
TG-heavy); без ввода крипто-адресов; UI оплаты берёт на себя Telegram.
**Минусы:** зависимость от Telegram; лёгкая привязка к TG-аккаунту.
**⚠️ Про Stars:** Stars покупаются пользователем через Apple/Google IAP → на этом шаге
теряется де-Google + комиссия сторов ~30%. **Поэтому в приоритете TON, а не Stars.**
**Анонимность:** средняя (TG-аккаунт, но без карты/имени); TON-оплата — выше.
**Комиссии:** TON — копейки (сеть); Stars — комиссия Telegram + сторов.

## 2. Вариант B — Криптовалюта через self-hosted BTCPay Server  [макс. анонимность]

**Как работает:**
- «Купить лицензию» → приложение просит бэкенд создать инвойс → BTCPay возвращает
  адрес/QR + сумму + `invoice_id` (в metadata кладём `session_token`/pubkey).
- Приложение показывает экран оплаты: QR + адрес + сумма + таймер + выбор монеты.
  ⚠ ПОПРАВКА (после ресёрча, детали в `BTCPAY_DETAILED.md`) — BTCPay реально умеет:
  - **Bitcoin (on-chain + Lightning)** — нативно; Lightning для мгновенной оплаты.
  - **Monero (XMR)** — максимальная анонимность, но через community **beta-плагин** и
    с **hot-кошельком** (риск при взломе сервера, в отличие от BTC).
  - **❌ USDT-TRC20 и TON — BTCPay НЕ поддерживает** (подтверждено). TON закрыт ботом
    Этапа 1; USDT-TRC20 — при необходимости отдельной рельсой (NOWPayments, non-custodial).
- Пользователь платит из своего кошелька → BTCPay ловит платёж → webhook на бэкенд →
  активация лицензии → приложение подхватывает статус.

**Клиент:** экран инвойса (QR, адрес, сумма, обратный отсчёт, выбор монеты) + поллинг.
**Сервер:** self-hosted **BTCPay Server** (свои кошельки, XMR-нода) + endpoint создания
инвойса + webhook → выдача лицензии. (Быстрый старт-альтернатива: NOWPayments/
CoinPayments — сторонний процессор, менее приватно, но без своей инфраструктуры.)

**Плюсы:** высшая анонимность (особенно XMR); полный суверенитет (self-hosted, без
третьих сторон, нет логов у процессора); идеально по духу продукта.
**Минусы:** нужно поднять и обслуживать BTCPay + ноды; у пользователя должна быть
крипта; подтверждения (XMR ~2 мин; BTC — Lightning для мгновенно); волатильность курса
(инвойс фиксирует курс на ~15 мин); обработка «застрявших»/недоплаченных инвойсов.
**Анонимность:** высшая.
**Комиссии:** только сетевые (нет процессора при self-hosted).

## 3. Вариант C — Коды/ваучеры (уже есть, автоматизировать продажу)

**Как работает:**
- Коды продаются через каналы (TG-магазин-бот, крипто-обменники, реселлеры).
  Пользователь покупает код как угодно и вставляет в приложение (текущий
  `activate-promo`). Продажу можно автоматизировать «магазином»-ботом (оплата TON/
  крипта → выдача кода).

**Клиент:** ничего не надо (ввод кода уже есть).
**Сервер:** минимум (код-система есть) + опционально бот-магазин.

**Плюсы:** максимально гибко и анонимно (код отвязан от платежа — можно купить за что
угодно, подарить, перепродать); уже работает на клиенте; идеально для реселлеров и
подарочных кодов.
**Минусы:** не «в приложении» (лишний шаг: купить код где-то → вставить).
**Анонимность:** высшая (полная развязка платежа и кода).

## 4. Вариант D — Фиат/карты  [НЕ рекомендуется для концепции]

Карты/фиат через процессор технически возможны, но тянут KYC, данные карты, привязку
к личности — ломают анонимность. Упомянут для полноты; для приватного мессенджера
не советую.

## 5. Сравнение

| Критерий | A. TG-бот (TON) | B. Крипта/BTCPay | C. Коды |
|---|---|---|---|
| Простота для юзера | ★★★★★ | ★★★☆ | ★★★ |
| Скорость запуска | быстро | средне (поднять BTCPay) | уже есть |
| Анонимность | средняя–высокая | высшая (XMR) | высшая |
| Суверенность (без 3-х сторон) | нет (Telegram) | да (self-hosted) | да |
| Комиссии | сетевые (TON) / Telegram (Stars) | только сетевые | зависит |
| Работа сервера | бот + webhook | BTCPay + webhook | минимум |

## 6. Рекомендация — поэтапно

**Этап 1 (быстрый MVP «оплата в приложении»):**
- **TON через TG-бот** (Вариант A) как основной быстрый путь.
- Оставить **коды** (Вариант C) как запасной/реселлерский.
- Уже это даёт «купить не выходя из мира приложения» + анонимно.

**Этап 2 (максимум приватности):**
- **Крипта через BTCPay** (Вариант B): XMR + BTC-Lightning + USDT/TON. Для тех, кому
  нужна высшая анонимность или нет Telegram.

**Этап 3 (полировка):**
- Авто-активация через `session_token` (без ручного кода), тарифы (1 мес / 6 мес /
  бессрочно), обработка недоплат/просрочек инвойса, курс/локализация цен.

Во всех этапах бэкенд-контракт один: оплата → активация лицензии для pubkey; клиент
лишь показывает статус.
