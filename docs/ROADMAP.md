## Тема: «воронка до покупки» — доступ без лицензии + покупка в приложении

**Проблема сейчас:** экран лицензии (`license_screen.dart`) даёт ТОЛЬКО ввод кода
(`/api/activate-promo`). Поддержка (`support_chat_screen`) и помощь (`help_screen`)
доступны только ИЗ НАСТРОЕК — то есть ПОСЛЕ активации. Значит:
- если код не проходит — человек заперт, написать некому;
- почитать о продукте перед покупкой негде;
- купить лицензию из приложения нельзя (только вставить готовый код).

Цель: закрыть всю воронку «узнал → написал если проблема → купил» прямо в приложении,
не ломая концепцию приватности/анонимности.

---

### 1. Поддержка ДО активации (без лицензии)  [клиент ✅ / сервер — проверить]

> ✅ **КЛИЕНТ СДЕЛАН:** на экране лицензии внизу (под «код не принят?») добавлена
> кнопка поддержки → открывает существующий `SupportChatScreen`. Идентификация в
> `SupportChatService` идёт по **pubkey** (`X-Pubkey`), не по лицензии, а pubkey на
> экране лицензии уже есть.
> ⚠ **СЕРВЕР — проверить:** endpoints `/api/support/message` (+ `/messages`,
> `/unread`, `/logs`) должны принимать запрос от pubkey **без активной лицензии**
> (pre-auth). Если сейчас они гейтятся лицензией — снять гейт для support.

**Что:** на экране лицензии добавить «Не проходит активация? Написать в поддержку».

**Зачем:** человек купил лицензию, а она не активируется — он должен мочь быстро
написать нам и получить помощь, не имея аккаунта. Сейчас такой путь тупиковый.

**ВАЖНО — механизм УЖЕ ЕСТЬ:** в Настройках есть пункт «Написать разработчику» →
сообщение сразу летит в админку (`support_chat_screen`). То есть с нуля строить НЕ
надо — задача в том, чтобы вывести ЭТОТ ЖЕ путь на экран лицензии (до активации).

**Как:**
- Добавить на `license_screen` кнопку «Написать разработчику» → открыть тот же
  `support_chat_screen` (или его облегчённую версию) в НЕавторизованном режиме.
- Убедиться, что серверный приём сообщения в админку работает БЕЗ активной лицензии
  (pre-auth): приложить введённый код + модель/версию к сообщению, чтобы мы сразу
  видели контекст проблемы с активацией.
- (Опционально) параллельно диплинк на TG-канал/бот поддержки как быстрый резерв.

**Тех. заметки:** переиспользовать существующий путь «Написать разработчику»; проверить/
доработать серверный endpoint на pre-auth + rate-limit.

### 2. Раздел «О продукте» ДО активации  [приоритет: высокий]

**Что:** на экране лицензии добавить «О продукте» / «Зачем Orpheus» → информационный
экран, доступный ДО покупки.

**Зачем:** дать почитать о продукте перед покупкой лицензии — повышает конверсию и
снижает нагрузку на поддержку.

**Как:** новый экран `about_screen` (или расширить `help_screen`), доступный с
`license_screen`. Контент простыми словами: что такое Orpheus, ключевое (E2E-
шифрование, независимость от Google, без номера телефона, звонки, заметки, PIN/duress/
паник-вайп), почему платный, как купить. Локализация RU/EN. Можно переиспользовать
материал из наших отчётов «было/стало».

**Тех. заметки:** статический контент (l10n), никаких сетевых зависимостей — грузится
даже без активации/сети.

### 3. Покупка лицензии В ПРИЛОЖЕНИИ  [приоритет: высокий]

> ✅ **РЕШЕНИЕ (принято владельцем):** self-hosted **USDT-TRC20 (TRON)** — HD-кошелёк
> генерит свежий адрес на каждый инвойс, свой вотчер чейна, при подтверждении
> активирует премиум. **In-app, без мессенджеров** (Telegram = конкурент, отпал),
> без посредников. Стейблкоин = без волатильности. Детальное ТЗ: **`USDT_TRON_SPEC.md`**.
> Ниже — история вариантов (для контекста). TON-бот отклонён (зависимость от Telegram).

**Что:** купить лицензию прямо в приложении — просто и быстро, в концепции
анонимности/безопасности.

**Ограничения концепции:**
- Приложение — прямой APK, НЕ Google Play → Play Billing отпадает.
- Де-гуглизация + приватность → без карт/идентификации/Apple-Google IAP где возможно.
- Бэкенд уже умеет выдавать/активировать коды (`/api/activate-promo`) — платёж нужно
  лишь свести к «оплатил → выдали лицензию».

**Варианты (проработаны):**

**Вариант A — Криптовалюта через self-hosted процессор**
- Монеты: **Monero (XMR)** — максимальная анонимность; + Bitcoin (можно Lightning для
  скорости), USDT (TRC-20), **TON** (быстро/дёшево).
- Процессор: **BTCPay Server (self-hosted)** — своя инфраструктура, без третьей
  стороны; идеально по концепции суверенности. (Альтернатива быстрого старта:
  NOWPayments/CoinPayments — сторонний, менее приватно.)
- Флоу: «Купить лицензию» → выбор монеты → инвойс/QR + сумма → оплата из своего
  кошелька → бэкенд ловит платёж (webhook BTCPay) → авто-выдаёт лицензию.
- Плюсы: высшая анонимность (XMR), полный контроль, в духе продукта.
- Минусы: крипта есть не у всех; подтверждения (XMR ~2 мин; BTC — Lightning для мгновенно).

**Вариант B — Telegram Stars / TON через бота**  (самый быстрый UX)
- Флоу: приложение открывает диплинк на наш TG-бот → покупка лицензии за **Telegram
  Stars** или **TON** прямо в Telegram → бот выдаёт код / бэкенд авто-активирует →
  возврат в приложение.
- Плюсы: очень просто/быстро; аудитория приватных мессенджеров TG-heavy; без ввода
  крипто-адресов. TON — нативно анонимно и дёшево.
- Минусы: привязка к Telegram; Stars покупаются через Apple/Google IAP (тут де-Google
  частично теряется) — поэтому предпочтительно **TON**, а не Stars.

**Вариант C — Готовые коды/ваучеры (уже есть, автоматизировать продажу)**
- Реселлеры / крипто-обменники / TG-каналы продают коды; человек вставляет код
  (текущий `activate-promo`).
- Плюсы: гибко, максимально анонимно (код за что угодно). Минусы: не «в приложении».

**РЕКОМЕНДАЦИЯ (комбо):**
1. **TON через TG-бот** — как основной «быстро и просто» способ в приложении.
2. **Крипта (XMR + BTC/USDT) через BTCPay** — для тех, кому нужна максимальная
   анонимность / нет Telegram.
3. **Коды** — оставить для реселлеров/поддержки/подарков.

Единый бэкенд-контракт: любой оплаченный путь сводится к выдаче/активации лицензии
(как `activate-promo`), клиент лишь показывает статус «активна».

**Открытый вопрос для решения владельца:** с чего начинаем — TON-бот (быстрый MVP) или
крипта/BTCPay (максимум приватности)? От этого зависит порядок работ клиента и сервера.
