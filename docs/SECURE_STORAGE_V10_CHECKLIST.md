# flutter_secure_storage 9 → 10: девайс-чеклист

Апгрейд `flutter_secure_storage` `^9.0.0` (9.2.4) → `^10.3.1` с **честным переходом
на современные шифры** (RSA-OAEP + AES-GCM).

`flutter_secure_storage` хранит невосстановимые секреты: приватный X25519-ключ
(личность), хэши PIN/duress/wipe + соль, и ключ шифрования БД (SQLCipher).

## Что сделано (без костылей)

- Используем СОВРЕМЕННЫЕ шифры v10 по умолчанию (OAEP + GCM) — не deprecated,
  живут в v11. Опции — единые для всех мест: `lib/services/secure_storage_options.dart`
  (`appSecureStorage`), `resetOnError: false`, `migrateOnAlgorithmChange: false`.
- Данные v9 (старые шифры PKCS1 + CBC) **осознанно приносятся в жертву** (закрытая
  бета). Авто-миграцию v10 НЕ используем — она ненадёжна (краши/частичное стирание,
  issues #1043/#1079). Вместо неё — детерминированный одноразовый `deleteAll()` ДО
  первого чтения (`ensureSecureStorageMigrated()` в `main.dart`, флаг
  `secure_storage_v10_reset_done` в SharedPreferences).
- `minSdk` 23 (v10 требует 23) — без изменений. API read/write/delete не менялся.

## ПОСЛЕДСТВИЕ (принято владельцем)

Существующие бета-пользователи после обновления **теряют старый аккаунт**
(личность/переписку) и создают новый. Это ожидаемо и согласовано.

## Критический тест — путь обновления v9 → v10 (не uninstall!)

- [ ] Собрать APK с ПРЕДЫДУЩЕГО коммита (v9.2.4), установить на реальное устройство, создать аккаунт + PIN + пару сообщений.
- [ ] **Обновить ПОВЕРХ** (не удаляя) на APK с этим апгрейдом (v10). То же устройство.
- [ ] После обновления проверить:
  - [ ] Приложение **открывается без краша** (главное — авто-миграция v10 не должна крашить старт; наш `deleteAll` до чтения это гарантирует).
  - [ ] Старого аккаунта нет — показан экран приветствия (данные v9 чисто стёрты, как и задумано).
  - [ ] **Создание нового аккаунта** работает: генерация ключей, задание PIN/duress/wipe.
  - [ ] Отправка/приём сообщений работает (новый ключ БД в современном формате пишется/читается).
  - [ ] **Перезапуск приложения**: новый аккаунт на месте, вход по PIN работает (современные шифры читаются повторно).
  - [ ] **panic-wipe / wipe-код** срабатывает (`deleteAll`).

## Чистая установка (новый пользователь)

- [ ] Свежая установка v10 (без данных v9): создание аккаунта, PIN, duress, сообщения, wipe — всё с нуля на современных шифрах.
- [ ] Перезапуск приложения и **перезагрузка устройства**: аккаунт и вход по PIN на месте (у v10 исторически ломалось «чтение после boot» — проверить).
